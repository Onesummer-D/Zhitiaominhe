# 智调民和 —— 民族地区基层纠纷智能调解与法律适配系统

> 面向少数民族边疆地区的通用基层纠纷智能调解平台。通过"规则引擎 + DeepSeek大模型语义兜底"双引擎融合，实现纠纷智能识别、习惯法冲突检测、法律文书一键生成与案件全生命周期管理。

---

## 项目背景

我国少数民族地区长期存在"国家法"与"民族习惯法"并行的二元治理格局。基层调解员在处理彩礼返还、土地边界、欠薪讨薪、邻里冲突等纠纷时，常面临习惯法效力不明、法律依据检索困难、调解协议缺乏规范文书支撑等痛点。

**智调民和** 基于 React + FastAPI 前后端分离架构，构建"国家法律库 + 民族习惯法库"双层知识库，为基层调解员提供从纠纷录入、AI分析、调解方案生成到诉讼准备的全流程智能化辅助工具。

---

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| **纠纷智能识别** | 双层级权重规则引擎（核心词+3 / 特征词+1 / 排除词-2），高分直接返回，低分/无匹配走 DeepSeek API 语义兜底 |
| **习惯法冲突检测** | 自动检索民族习惯法库，标注与国家法律的冲突点及效力优先级 |
| **调解路径指引** | 三段式路径：非诉调解 → 司法确认 → 诉讼，生成结构化调解建议 |
| **法律文书生成** | 支持人民调解协议书、民事起诉状、检察监督申请书 3 类文书自动生成，符合正式文书排版规范 |
| **案件全生命周期管理** | 待调解 / 调解中 / 已结案 / 调解失败 状态流转，支持查看回填与历史回溯 |
| **知识库检索** | 国家法律库 + 民族习惯法库（彝族"德古"、哈尼族"咪谷"等）动态检索 |

## 技术架构

### 前端
- React 18 + React Router DOM 6（多页面路由）
- Axios HTTP 请求
- 民族风视觉设计：主色 #2C7473 深青绿 / 强调色 #A85A34 赭石

### 后端
- FastAPI + Uvicorn（ASGI）
- SQLite3 案件数据存储
- Python-jieba 中文分词（规则引擎）
- DeepSeek API 语义理解与兜底分析

## 快速开始

### 环境要求
- Node.js ≥ 18
- Python ≥ 3.10
- DeepSeek API Key（通过环境变量 `DEEPSEEK_API_KEY` 配置）

### 启动前端
```bash
cd frontend
npm install
npm start
# 默认运行在 http://localhost:3000
```

### 启动后端
```bash
cd backend
pip install -r requirements.txt
# 配置环境变量：DEEPSEEK_API_KEY=your_key_here
python main.py
# 默认运行在 http://localhost:8000
```

## 核心算法

**双层级权重计分（置信度计算）**
- 核心定性词命中 +3 分（如"家暴""欠薪""侵占""彩礼"）
- 特征词命中 +1 分（如"打""欠""占""退"）
- 排除词触发 -2 分（如"做梦""假设""听说"）

| 置信度 | 分数 | 处理策略 |
|-------|------|---------|
| 高 | ≥ 6 分 | 直接返回规则引擎结果 |
| 中 | 3-5 分 | 规则引擎 + DeepSeek API 二次确认 |
| 低 / 无匹配 | < 3 分 | 完全交由 DeepSeek API 语义兜底 |
